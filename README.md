# board
전역적인 경로나 기능을 다루는 라우터를 직접 생성

나머지 기능들에 대한 라우터는 routes/index.js를 통해서 가져오기
1. 사용자 인증
2. 에러처리
3. 정적파일 라우터
4. API버전 라우터
5. pw 해쉬화 진행하기 (써보는데 의의)

개인과제를 진행하며 여러가지 기능들을 구현해 보고 싶어서, 레퍼런스를 참조하며 어떤 방식으로 구현할지 정리했다.
기본 기능을 구현하는데는 오랜 시간이 걸리지 않았지만, 로컬호스트에 저장을하고 계속 서버를 통해서 get을 요청하는 바보짓을 했다.

그저, 가볍게 몽고db에 저장하면 로컬호스트와 서버의 ip 호스트가 동시에 참조할 수 있을 것 같았으며, 의심또한 하지 않았다.

또한, 학습을 했음에도 불구하고 몽구스의 모델과, 몽구쉬 등 대화를 나눴을 때 바로바로 워딩이 떠오르지 않아 당황스러웠다.
물론 밤을세운 영향도 있겠지만 협업을 하기 위해서는 이러한 워딩을 확실하게 머릿속에 넣었을 때 적어도 답답함은 없이 소통할 수 있을 것 같다는 생각이 들었다.
